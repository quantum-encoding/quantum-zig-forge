# Migration Card: std/Random/Sfc64.zig

## 1) Concept

This file implements the Sfc64 (Small Fast Counting) pseudo-random number generator, which is one of the fastest PRNG engines from the Practically Random (PracRand) test suite. The implementation provides a 64-bit random number generator with a small state footprint consisting of four 64-bit integers (a, b, c, counter). The key public interface allows seeding the generator and obtaining random bytes through a `std.Random` compatible interface.

The core functionality includes state initialization with a seed, generation of random 64-bit values via the internal `next()` method, and buffer filling for byte-oriented random data. The implementation uses rotation, shifting, and modular arithmetic operations to maintain the generator's statistical properties while providing high performance.

## 2) The 0.11 vs 0.16 Diff

**No significant migration changes detected** - This PRNG implementation follows consistent patterns between Zig 0.11 and 0.16:

- **No explicit allocator requirements**: The `init()` function returns a struct directly without heap allocation
- **Consistent I/O patterns**: The `fill()` method maintains the same signature for buffer population
- **No error handling changes**: All operations are infallible (return `void` or direct values)
- **Stable API structure**: The factory pattern (`init()`) and method signatures remain unchanged

The public API maintains backward compatibility:
- `init(u64) Sfc64` - Factory function with seed parameter
- `random() std.Random` - Returns std.Random interface
- `fill([]u8) void` - Buffer filling method

## 3) The Golden Snippet

```zig
const std = @import("std");
const Sfc64 = std.Random.Sfc64;

// Initialize with seed
var prng = Sfc64.init(0x123456789ABCDEF);

// Get std.Random interface
var random = prng.random();

// Generate random values using standard interface
const random_int = random.int(u64);
const random_float = random.float(f64);

// Or use fill directly for byte-oriented operations
var buffer: [16]u8 = undefined;
prng.fill(&buffer);
```

## 4) Dependencies

- `std` - Core standard library
- `std.math` - For rotation operations (`rotl`)

**Note**: This is a self-contained PRNG implementation with minimal dependencies, primarily using mathematical operations from `std.math` for bit manipulation.
