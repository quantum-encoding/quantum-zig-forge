# Migration Card: `std/os/linux/mipsn32.zig`

## 1) Concept

This file provides low-level Linux system call implementations and type definitions specifically for the MIPS n32 architecture. It contains inline assembly implementations of system call functions (`syscall0` through `syscall6`, `syscall_pipe`, and `clone`) that handle the MIPS n32 calling convention and error handling. The file also defines architecture-specific type aliases (like `blksize_t`, `off_t`, etc.) and a `Stat` structure for file system metadata with MIPS n32-specific field layout and padding.

Key components include:
- Raw system call functions with varying numbers of arguments
- Specialized implementations for `pipe` and `clone` system calls
- MIPS n32 ABI-specific type definitions and structure layouts
- VDSO (Virtual Dynamic Shared Object) constants for optimized system calls

## 2) The 0.11 vs 0.16 Diff

**No significant migration changes detected for public APIs.** This file contains architecture-specific low-level system call implementations that follow consistent patterns:

- **System Call Functions**: All syscall functions (`syscall0`-`syscall6`, `syscall_pipe`, `clone`) maintain the same signature pattern - taking system call numbers and raw integer arguments, returning `u32` results
- **No Allocator Changes**: These are raw system call wrappers that don't involve memory allocation
- **No I/O Interface Changes**: These are direct kernel interface functions, not high-level I/O abstractions
- **Error Handling**: Consistent use of MIPS-specific error return convention (negating negative error codes)
- **API Structure**: No init/open pattern changes - these are direct system call wrappers

The primary differences are internal implementation details in the inline assembly, particularly the use of `@intFromEnum(number)` instead of direct integer values, but the public API signatures remain unchanged.
