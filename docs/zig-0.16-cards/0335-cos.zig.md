# Migration Card: `std/math/complex/cos.zig`

## 1) Concept
This file implements complex cosine functionality in Zig's standard library. It provides a public `cos` function that calculates the cosine of a complex number using the identity relationship with hyperbolic cosine. The implementation transforms the input complex number and delegates to the existing `cosh` implementation.

Key components:
- Public `cos` function that accepts any complex number type
- Mathematical transformation using complex number properties
- Test coverage with specific expected values

## 2) The 0.11 vs 0.16 Diff

**Explicit Allocator Requirements**: No allocator needed - this is a pure mathematical function that operates on complex numbers without memory allocation.

**I/O Interface Changes**: No I/O dependencies - operates purely on mathematical types.

**Error Handling Changes**: No error handling - this is a deterministic mathematical function.

**API Structure Changes**:
- Uses `Complex(T).init()` pattern for complex number construction
- Generic type inference using `@TypeOf(z.re, z.im)` for return type compatibility
- Maintains functional purity with no side effects

## 3) The Golden Snippet

```zig
const std = @import("std");
const math = std.math;
const complex = math.complex;

const a = complex.Complex(f32).init(5, 3);
const result = complex.cos(a);
// result.re ≈ 2.8558152, result.im ≈ 9.606383
```
