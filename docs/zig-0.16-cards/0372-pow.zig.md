# Migration Card: std/math/pow.zig

## 1) Concept

This file implements a mathematical power function (`x^y`) for floating-point and integer types in Zig's standard library. It's a port from Go's math library and provides a comprehensive power calculation with extensive special case handling for IEEE 754 floating-point standards. The key public API is the `pow` function that takes a type parameter and two values, returning the result of raising the first value to the power of the second.

The implementation handles numerous edge cases including zero, infinity, NaN, and negative exponents, with specific behaviors defined for each combination. It includes optimizations for common cases like square roots and uses internal helper functions for integer power calculations and odd integer detection.

## 2) The 0.11 vs 0.16 Diff

**No significant API migration changes detected** for this mathematical utility function:

- **No allocator requirements**: This is a pure mathematical computation that doesn't require memory allocation
- **No I/O interface changes**: The function operates purely on numeric inputs/outputs
- **No error handling changes**: The function doesn't return error unions - it handles all cases internally
- **API structure remains consistent**: The function signature `pow(comptime T: type, x: T, y: T) T` follows standard mathematical utility patterns

The function maintains the same generic type parameter approach that was common in Zig 0.11 for mathematical utilities. It delegates to `math.powi` for integer calculations and handles floating-point cases directly.

## 3) The Golden Snippet

```zig
const std = @import("std");
const math = std.math;

// Calculate 2.0 raised to the power of 3.0
const result = math.pow(f64, 2.0, 3.0);
// result = 8.0

// Calculate square root using power function
const sqrt_result = math.pow(f64, 16.0, 0.5);
// sqrt_result = 4.0

// Handle special cases automatically
const zero_power = math.pow(f64, 5.0, 0.0);
// zero_power = 1.0 (any number to power of 0 is 1)
```
